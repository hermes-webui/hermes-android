# Projects

This folder is for active or planned project work that needs more detail than a single issue comment.

Each project folder should stay focused, uncluttered, and tied back to the GitHub issue or discussion that started the work.

## Guidelines

Each project folder should include:

- a short `README.md`
- the related GitHub issue link
- a roadmap or checklist
- any design notes needed before implementation

Keep repo-wide docs in the root docs unless the information only applies to one project.
